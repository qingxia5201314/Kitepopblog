# 移动端 UI 修复与记账页优化设计

## 目标

在不改变网站总体风格、首页排版和记账业务逻辑的前提下，完成四项移动端优化：

1. 修复首页文章筛选下拉菜单被文章区域覆盖的问题。
2. 优化记账页的移动端排版、流水信息展示、按钮和动效。
3. 修复后台、文件、图床、记账四个口令入口的移动端排版。
4. 让顶部“工具”菜单在点击菜单外部、选择菜单项或按下 `Esc` 后自动收起。

## 设计边界

- 首页现有布局、卡片、间距、内容顺序和视觉风格保持不变，只修复筛选菜单层级。
- 网站继续使用现有米白、SOS 红、暖色纸张感和日系视觉语言，不换肤。
- 记账的数据结构、接口、字段、校验、筛选、增删改和预算计算逻辑保持不变。
- 不增加设备性能检测或低性能自动降级；默认使用完整视觉效果。
- 不引入大面积紫粉渐变、Emoji 分类宫格、巨型余额营销卡、实物装订本、横线纸、印章、撕页等效果。
- 删除操作界面中的教学式、营销式和动效解释文案。
- 动效只增强操作反馈和空间层级，不改变数据结果或请求顺序。

## 方案概览

采用“局部视觉重排 + 少量纯 UI 状态”的实现方式：

- 保留现有业务 hooks、API 调用和表单事件处理。
- 首页筛选器通过层叠上下文和浮层定位修复覆盖问题。
- 记账页只增加移动端分区页签状态，默认打开“记一笔”。
- 工具菜单改为受控开合，以支持可靠的外部点击关闭。
- 四个口令入口共享移动端排版规则，不合并或改写各自的认证逻辑。

## 1. 首页文章筛选器

### 行为

- 保持现有首页 DOM 顺序和视觉排版。
- 筛选菜单展开后形成独立浮层，始终位于文章列表和文章菜单之上。
- 浮层使用现有背景、边框、阴影和圆角，不新增遮罩或重排下方内容。
- 菜单展开和收起使用短距离淡入与位移，不推动文章卡片。

### 技术边界

- 优先修复产生问题的 `position`、`z-index`、`overflow` 和 stacking context。
- 不通过给首页整体设置异常大的全局 `z-index` 规避问题。
- 不修改筛选 URL、查询条件和文章加载逻辑。

## 2. 记账页移动端

### 信息架构

移动端采用四个展示分区：

- 概览
- 记一笔（默认）
- 流水
- 计划

页签只是展示状态。原有概览指标、记账表单、流水筛选与列表、预算和存钱计划仍使用现有数据与处理函数。桌面端保持现有信息结构，重点优化范围为移动端。

### “记一笔”分区

- 保留现有字段：收支类型、金额、分类、日期、支付方式、自定义分类、备注、计入存钱项目。
- 保留编辑流水时的现有字段回填和保存更新逻辑。
- 收支切换使用切角分段按钮。
- 金额输入保持最强视觉层级，但不制作巨型营销卡。
- 分类、支付方式、日期和备注使用两列紧凑网格；窄屏不足时允许单列。
- 自定义分类使用风格化描边展开按钮，内部管理功能保持原样。
- “计入存钱项目”保留原有说明，因为它解释真实计算规则；不添加额外教学文案。
- 保存按钮使用 SOS 红切角样式、错位底影和短按下反馈。

### 保存按钮动效

- 按下反馈沿用短时 `:active` 下沉，不增加长时间提交动画。
- 白色高光条循环横向扫过按钮。
- 高光位置按按钮容器宽度计算，从左外侧 `-20%` 移动到右外侧 `115%`，确保完整扫过后再淡出。
- 高光不使用基于自身宽度的 `translateX(%)` 作为主要位移，避免中途重置。

### 流水分区

- 使用现代纸张式卡片，不使用实体记账本、装订环或横线纸。
- 每条流水包含收支标识、标题、创建时间、存钱项目状态、金额、编辑和删除操作。
- 支出和收入通过左侧色条、标识底色和金额颜色区分。
- 列表进入使用逐项淡入和短距离位移；环境粒子沿用网站色板。

### 流水文本规则

- 时间只展示 `createdAt`。
- 编辑后仍展示原创建时间，不拼接 `spentAt`，不显示“发生”字样。
- 普通分类标题格式为：`分类名称 · 支付方式`。
- 分类为“其他”且备注非空时，标题格式为：`备注内容 · 支付方式`。
- 分类为“其他”且备注为空时，回退为：`其他 · 支付方式`。
- 日期字段仍保留在录入和编辑表单中，后端数据也保持不变；这里只改变流水列表的展示文本。

### 流水按钮

- 编辑按钮：深色细描边、金色识别短线、轻微上浮和按压反馈。
- 删除按钮：SOS 红细描边、右上切角、轻微上浮和按压反馈。
- 两类按钮均保持足够触控面积和清晰焦点状态。
- 删除确认和实际删除逻辑保持现状。

## 3. 四个口令入口

覆盖以下页面：

- 后台
- 文件仓库
- 图床
- 记账中心

### 移动端布局

- 使用现有 `unlock-panel` 视觉语言，不改成新的主题卡片。
- 在安全区内垂直居中，外侧保留稳定页边距。
- 标题、说明、密码框和主按钮改为单列，控件宽度占满容器。
- 密码框与按钮满足移动端触控高度，避免横向挤压和溢出。
- 长说明文字限制行宽并改善行距；不新增重复说明。
- 只统一响应式结构和视觉细节，各页面继续使用自己的口令、接口和提示文本。

## 4. 工具菜单自动收起

### 开合规则

- 点击“工具”切换开合状态。
- 点击菜单外的任意空白区域后关闭。
- 点击任一工具链接后关闭。
- 按下 `Esc` 后关闭，并把焦点返回“工具”触发器。
- 路由变化后关闭，避免新页面保留展开菜单。
- 菜单内部点击不触发意外关闭。

### 可访问性

- 触发器提供 `aria-expanded` 和菜单关联信息。
- 保持键盘可操作和清晰的 `focus-visible` 状态。
- 菜单关闭时不保留可聚焦的隐藏链接。

## 动效与视觉规范

- 使用现有 CSS 色板，不创建独立霓虹主题。
- 粒子可明显漂浮，但不得遮挡输入框、按钮文字或流水金额。
- 浮层和卡片可使用轻量 3D 景深；正文、表单文字和输入光标不参与倾斜。
- 菜单展开以 180–240ms 为主，卡片和分区进入以 320–480ms 为主。
- 点击反馈短于页面转场，避免操作已完成但视觉仍被锁住。
- 保留项目已有的系统级无障碍行为，不新增设备性能自适应逻辑。

## 数据流与错误处理

- 所有记账请求继续经过现有 `useAccounting`、`accountingApi` 和服务端接口。
- 页签切换不发起额外请求；进入流水分区时复用当前已加载数据。
- 现有加载失败、口令错误、保存失败和删除失败提示保持不变。
- UI 动画失败或被浏览器跳过时，表单提交、路由跳转和菜单关闭仍必须正确完成。

## 测试与验收

### 首页筛选器

- 移动端展开筛选菜单时，其层级高于文章列表。
- 首页结构、筛选参数和文章加载行为没有变化。

### 记账页

- 移动端默认展示“记一笔”。
- 四个分区能切换，且不改变原有数据和表单状态。
- 所有现有记账字段仍可录入、编辑和保存。
- 流水只显示创建时间。
- “其他”分类分别覆盖有备注与无备注两种标题回退。
- 保存按钮高光从按钮左外侧完整移动到右外侧。
- 编辑、删除和保存按钮具备鼠标、触控和键盘反馈。

### 口令入口

- 四个页面在 320px、375px 和 430px 宽度下无横向溢出。
- 输入框、主按钮和说明文字均保持可读、可触控。

### 工具菜单

- 覆盖点击触发器、菜单内部、外部空白、工具链接、`Esc` 和路由变化。
- 验证焦点返回和 ARIA 状态。

### 全量验证

- 运行相关前端单元测试与组件测试。
- 运行完整测试套件和生产构建。
- 在桌面端确认首页和记账页没有非预期重排。
- 在移动视口手动检查层级、滚动、触控区域、粒子遮挡和按钮高光。
