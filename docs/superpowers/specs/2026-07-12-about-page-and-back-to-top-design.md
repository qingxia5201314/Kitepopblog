# “关于我”页面与回到顶部按钮设计

## 目标

为现有 Kitepop SOS 博客增加一个可由后台维护的“关于我”页面，并增加全站可用的回到顶部按钮。新页面应参考用户提供的居中式个人主页样板，同时优先延续当前博客的红、米白、黑色海报风与 SOS 视觉语言，并完整适配移动端。

## 范围

本次交付包含：

- 导航栏新增“关于我”入口。
- 新增公开路由 `/about`。
- 后台新增独立的“关于我”编辑模块。
- 新增关于我资料的 SQLite 持久化、公开读取接口和管理员更新接口。
- 复用现有图床完成头像上传。
- 新增全站回到顶部按钮。
- 覆盖桌面端与移动端的布局、交互、异常状态和自动化测试。

不包含工作或学习时间线、项目经历、多个社交平台、邮件联系方式、SEO 专用字段或关于我内容的版本历史。

## 数据模型

使用独立的 SQLite 单记录资料表，不把关于我内容伪装成普通文章，也不使用额外 JSON 配置文件。

`about_profile` 记录包含：

- `avatar_url`：通过现有图床上传后得到的头像地址。
- `display_name`：页面展示名称。
- `identity_tags`：多个短身份标签，以项目现有惯例支持的稳定序列化格式保存。
- `intro`：首屏简短介绍。
- `github_url`：GitHub 个人主页 HTTPS 地址。
- `content`：Markdown 详细介绍。
- `updated_at`：最近更新时间。

表在服务启动时按现有数据库初始化模式自动创建。资料尚未配置时，公开读取返回字段齐全的空资料，而不是 404 或服务端错误。

## 服务端接口

### 公开读取

`GET /api/about`

- 无需登录。
- 返回统一的 `{ profile }` 响应。
- 资料不存在时返回默认空资料。
- 可设置适合公开资料的短时缓存策略，但管理员保存后必须能在下一次前台请求中读取到最新值。

### 管理员更新

`PUT /api/admin/about`

- 复用现有管理员会话与 `requireAdmin` 中间件。
- 接收完整资料对象并进行服务端规范化与校验。
- 名称去除首尾空白；身份标签去重并限制数量与单项长度；介绍和 Markdown 正文限制最大长度。
- `github_url` 允许为空；非空时必须是 `https://github.com/...` 个人主页形式，拒绝凭据、脚本协议和非 GitHub 主机。
- 保存成功后返回规范化后的 `{ profile }`。
- 校验失败返回 400，未授权返回现有认证体系对应的状态码。

头像上传继续使用现有图床接口和限制，不新增重复的二进制上传接口。

## 前台“关于我”页面

### 路由与导航

- 在 `Layout` 的“首页”之后增加公开导航项“关于我”。
- `/about` 时显示现有导航激活态。
- 页面组件采用懒加载，与其他非首页页面保持一致。

### 首屏资料区

首屏保持参考图的居中个人资料结构，但使用本站视觉系统：

- 圆形头像位于视觉中心，带旋转虚线外圈和红色呼吸光晕。
- 名称、身份标签和简短介绍按清晰层级排列。
- 只显示一个 GitHub 圆形社交按钮。
- GitHub 链接在新标签页打开，并使用 `rel="noopener noreferrer"`。
- 未配置 GitHub 地址时不渲染按钮。
- 头像缺失或加载失败时，回退到现有品牌头像或基于名称的字母头像。
- 首屏背景加入倾斜、低透明度的大号 “SOS” 水印，以及较小的海报式英文标记。

### Markdown 内容区

- 复用现有 `MarkdownContent` 渲染能力和安全边界。
- 不启用原始 HTML 注入。
- 内容使用单列阅读宽度，标题分隔线、卡片边框和错位阴影延续本站海报风。
- 资料整体为空时显示友好的“内容尚未配置”状态；读取失败时显示错误信息和重试按钮。

## 动态效果

默认提供完整的动态体验：

- 页面载入时头像、标题、标签、简介和 GitHub 按钮依次淡入上移。
- 大号 “SOS” 水印由低透明度、轻微旋转状态进入，并持续做非常缓慢的漂移。
- 头像虚线外圈缓慢旋转，光晕做轻微呼吸。
- GitHub 按钮在桌面端悬浮时上移、改变边框颜色并出现海报式错位阴影。
- Markdown 内容区在进入视口时逐段显现。
- 桌面端首屏支持轻度指针视差；移动端不使用指针视差，改用触摸反馈。

持续动画和入场动画只改变 `transform`、`opacity` 等合成友好属性。系统开启 `prefers-reduced-motion: reduce` 时保留完整内容和状态反馈，但停止持续漂移、旋转、视差和平滑滚动，避免可访问性问题。

## 后台编辑模块

后台左侧管理区增加“关于我”折叠项，交互和视觉沿用现有文章、用户模块。打开后显示独立资料编辑面板，不复用文章发布状态、定时发布或版本历史。

编辑项包括：

- 头像上传按钮与即时预览。
- 名称输入框。
- 多身份标签输入。
- 简短介绍输入区。
- GitHub 个人主页链接输入框。
- Markdown 详细介绍编辑器及编辑/预览标签页。
- 保存按钮、加载状态和保存结果提示。

打开模块时读取现有资料。头像上传成功后只更新当前表单预览，资料仍在点击保存后统一提交。保存失败时保留所有输入，不重置表单；字段错误应显示在对应控件附近并可聚焦定位。

## 回到顶部按钮

回到顶部按钮作为 `Layout` 下的全站共享组件，因此首页、文章页、关于我页面与工具页面均可使用。

- 页面垂直滚动超过约 400px 时进入显示状态。
- 组件始终挂载，通过状态类控制可见性。
- 隐藏态使用 `opacity: 0`、轻微向下位移和 `pointer-events: none`。
- 显示态使用 `opacity: 1`、原始位置和可点击状态。
- 显隐过渡约 200–300ms，以透明度为主并搭配轻微位移。
- 点击后使用平滑滚动返回页面顶部；回到顶部过程中按钮随阈值状态逐渐淡出。
- 视觉采用红/米白色、海报线条、箭头旋转和轻量粒子反馈，与现有指针效果协调。
- 按钮提供明确的 `aria-label` 和键盘焦点样式。
- 移动端保持至少 44 × 44 CSS 像素点击区域，并使用 `env(safe-area-inset-bottom)` 避让底部安全区。

滚动监听使用被动事件或等效的低开销方案，并通过单一状态更新避免每帧重复渲染。

## 移动端适配

- 关于我首屏改为紧凑单列，缩小 SOS 水印、头像、标题和区块间距。
- 水印不得造成横向滚动，也不得压低正文对比度。
- 身份标签允许自然换行，GitHub 按钮保持足够触摸尺寸。
- Markdown 内容、代码块、表格和长链接不得撑破视口。
- 后台表单在窄屏按单列排列，编辑/预览切换和上传按钮保持可触达。
- 回到顶部按钮避开系统安全区和页面主要操作控件。

## 数据流

1. 访问 `/about` 时，前端调用公开接口读取资料。
2. 成功后渲染首屏结构化字段和 Markdown 正文；失败时提供重试。
3. 管理员打开后台“关于我”模块时，读取相同资料并填充表单。
4. 管理员上传头像，现有图床返回 URL，前端更新头像字段与预览。
5. 管理员点击保存，前端先做基础校验，再调用管理员更新接口。
6. 服务端重新校验、规范化并写入 SQLite，返回最终资料。
7. 前端以服务端返回值刷新表单并显示保存成功通知。

## 错误处理

- 公开读取失败：保留页面外壳，显示简洁错误状态和重试按钮。
- 后台读取失败：显示错误通知，不用空响应覆盖管理员当前输入。
- 头像上传失败：保留原头像字段和其他表单内容。
- GitHub 地址无效：前后端均拒绝保存，并给出可操作提示。
- 保存失败：保留表单，解除提交禁用状态，允许重试。
- Markdown 内容为空：允许保存并在前台仅展示资料首屏；名称为空时前端和服务端均阻止保存。

## 测试与验收

### 服务端

- 数据表首次初始化与重复初始化。
- 空资料的公开读取。
- 完整资料的读取与更新。
- 管理员接口拒绝未授权请求。
- 名称、标签、内容长度及 GitHub URL 校验。
- 数据持久化后重新打开数据库仍可读取。

### 前端

- 导航“关于我”的链接与激活态。
- 关于我页面加载、错误重试、空状态和完整资料渲染。
- 仅在存在有效 GitHub 地址时显示外链按钮。
- 头像回退行为。
- 后台资料读取、字段编辑、头像上传预览、Markdown 预览、保存成功与失败保留表单。
- 回到顶部按钮在阈值上下通过透明度状态显隐，隐藏态不可点击，点击触发返回顶部。

### 人工检查

- 桌面端核对首屏水印、视差、入场动画、Markdown 阅读宽度和按钮反馈。
- 移动端核对 320px 以上常见视口、标签换行、无横向溢出、触摸目标和安全区。
- 核对键盘导航、焦点可见性和系统减少动态效果设置。
- 运行现有测试套件与生产构建，确保无回归。

## 完成标准

- 访客能从导航进入 `/about` 并稳定查看个人资料。
- 管理员能在后台上传头像、编辑所有约定字段并持久化保存。
- 前台只展示一个可正确跳转的 GitHub 按钮。
- 页面视觉明显属于现有 Kitepop SOS 网站，同时保留参考样板的居中个人主页结构。
- 回到顶部按钮在全站按滚动阈值用透明度平滑显隐。
- 桌面端和移动端均无布局溢出或关键交互遮挡。
- 自动化测试和生产构建通过。
